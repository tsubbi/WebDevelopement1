No More Duplicated Code

This is part of the code of this week's lab3. In this example, HTML code is 27 line, and css file has aproximately 160 lines. For this mini page that uses 160 lines is too much and unnecessory.

See the Pen CSS-Orginal by Tsubbi (@tsubbi) on CodePen.

Let's look at the code

The first thing in this CSS code will notice, each square's style setting is very long, but at the end result, they all look the same. So maybe it is possible to have work on it. Dive in to the code, that will find out the only difference appears to be the difference in animation.

To solve these problems, bese way is to standerize the look so every square that can apply it other difference can write seperatly.

I wrote a standarized class, called "square" and let all squares to apply it. Including the standarized hover behavior. To seperate each difference, I also created different class for each square to adapt.

See the Pen CSS-Reduced by Tsubbi (@tsubbi) on CodePen.

The end result is a huge improvement that reduce the code from 160 down to 70 that is over half of the original number of code.

Conclusion

By standerizing the style will make it reusable and efficient. It reduce the data size transfered from server to client which lowered the bandwidth and improve loading speed. Please bare in mind, it is very Important to optimize the code that will help out in long run.
