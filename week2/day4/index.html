Why need code reduction?

Code Reduction or refactoring code is usually underlooked by beginner developer. The benifit gained from code reduction may seemed unnecessory at beginning, but in long run, it will serve a huge difference when maintaining a huge project and every developer will encounter this issue when facing a lagacy code.

Normally when working, it will not always the case to start a project from beginning and to work efficiently on the project, it is necessory to understand the source code. At this point, clean code will come to an effect. Keeping the clean as clean as possible is very important. This is also in the case for CSS.

Reasons:

  1. People generally have a low attention span. This also apply to developers
  2. End user would come back more if the application loaded quicker
  3. Faster websites are ranked higher up in Search Engines
  4. Faster websites add value to SEO stratagy
