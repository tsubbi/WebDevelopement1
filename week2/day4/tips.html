No More Duplicated Code

DON'T REPEAT YOURSELF

Reduce repetition is the philiosophy. This is a general software development principle and can be applied in any programming language, as well as in CSS. As we can understand from its name, DRY aims to avoid or reduce repetition as much as possible. DRY also helps out with bandwidth increase site's loading speed. Oppositing from DRY is WET, Wasting Everyon's Time / We Enjoy Typing.

Don't use inline-styles

The use of inline-styles is very tricky. Although it provides exceptions but over abusing it may cause difficulties in code maintaining. For example, if a website is using large amount of inline-styles, by having a small change, would take a while to search and very hard to read. Another downside for inline-styles is it will cause overwrite issue that might have conflict with external css.

Keep the design simple

It is the best to have reusable and repeatable design for developer to use more than once. It not only increases the efficiently of developing but also helps out let the website to have a consistant style though out the whole site.

Use Shorthands

Some of the CSS properties like paddings, margins, fonts and borders can be written in a much simpler way by shorthands. Using shorthands helps to reduce the lines of rules.

Break a large file into mini CSS files

By doing this, will prevent A style that not only focus on the css functions but also decupling between css styles.
